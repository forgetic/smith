# Documentation

Smith documentation follows Diátaxis so each document has a clear job and agents
can load only the context relevant to a task.

## Tutorials

Learning-oriented introductions. Smith has no full tutorials yet; see
`tutorials/` for the placeholder index.

## How-to guides

Task-oriented recipes. Start with:

- `how-to/start-a-development-session.md`
- `how-to/fast-local-iteration.md`
- `how-to/end-a-development-session.md`

Provider and Temper-integration recipes live in `how-to/`.

## Reference

Precise contracts and factual details. Start with
`reference/development-conventions.md`, then read task-relevant pages such as
`reference/provider-auth.md`, `reference/process-responders.md`, and
`reference/testing.md`.

Smith-owned lessons live in `reference/agent-lessons/`.

## Explanation

Conceptual material and rationale. Start with
`explanation/process-boundary.md` for the Temper/Smith ownership model.

## Document size budget

Keep hand-written docs focused: aim for about 150 lines or fewer, and split a
page before it exceeds about 350 lines.

## Architecture decision records

Decision records capture significant choices that should remain visible to future
agents. See `adr/`.
